# agentic-go-cli

A minimal code-editing agent that, instead of re-sending an ever-growing message
history, keeps only the most recent Turns in full and collapses each older Turn to a
one-line Description on disk — letting the model recall a full transcript on demand
and forget what it no longer needs.

## Context management

**Artifact**:
A non-reproducible tool output — a `run` command's result (test runs, git output,
network calls). Unlike a read, which is a live reference re-read from the working tree
on demand, a run output cannot be reproduced, so it is preserved in its Turn's
transcript and recalled with the Turn — never as an independent entry.
_Avoid_: tool result, output file, blob

**Manifest**:
The conversation's abbreviated tail sent to the model in place of full content: one
Description per collapsed Turn. The most recent N Turns are sent in full; every older
Turn is a single Description — pointing at its transcript — until recalled. The Turn
is the only unit here; a Turn's tool calls fold into its Description rather than
appearing on their own.
_Avoid_: index, catalog, listing, 清单, working set, context, window, history

**Description**:
The one line that stands in for a collapsed Turn, by which the model decides whether
to recall its full transcript. It is generated by an LLM call at the moment the Turn
collapses — reading the whole Turn to capture the outcome and the key facts a future
round would need, not merely the request. The user prompt's first line is only the
fallback if that call fails; the model can refine any Description later via describe.
_Avoid_: summary, label, caption

**Forget**:
The model's action of dropping a collapsed Turn from the Manifest entirely, so its
Description stops being sent at all. The inverse is letting the Turn rest as its one
line, recall-able on demand.
_Avoid_: delete, prune, evict

**Recall**:
The model's action of pulling a collapsed Turn's full detail back into context by
reading its transcript (`turns/NNN.md`) — the complete exchange, including any
non-reproducible run output. The inverse of letting it rest as just its Description.
_Avoid_: reload, fetch, expand, rehydrate

## Session layout

**Session**:
One conversation, from the first user prompt to exit, persisted as a unit and
resumable later. Bare startup resumes the most recent Session; `--new` starts a
fresh one.
_Avoid_: chat, run, conversation log

**Turn**:
One user prompt plus the entire AI reply that follows it — however many model
responses and tool calls that takes — up until the agent asks the user for input
again. The unit of memory: persisted as a per-Turn transcript on disk, sent in full
while within the recent-Turns window, and collapsed to a single Description once it
falls outside — its tool calls folding into that line. A collapsed Turn is
recall-able via its transcript and forgettable.
_Avoid_: exchange, conversation, interaction

**Round**:
A single model response within a Turn (one call to the Messages API) plus the tool
results it triggers. The unit the Manifest is rebuilt for. (The in-memory counter
named `turn` today is really a Round counter.)
_Avoid_: step, iteration, request, turn
